# Ukázkový review projekt

## Role týmu Vector Forge
- **Filip** — lidský ředitel
- **Astra** — provozní manažerka
- **Daedalus** — systémový architekt
- **Forge** — vývojový inženýr
- **Sentinel** — recenzent kódu
- **Hydra** — bezpečnostní / chaos inženýr
- **Mnemosyne** — archivářka znalostí
- **Prometheus** — výzkum a inovace

## Konvence psaní kódu
- Styl pojmenování: **lowerCamelCase** (Lazarus/FPC)
- Prefixy:
  - lokální proměnné: `p`
  - fieldy: `f`
  - globální proměnné: `g`
  - parametry: `a`
- Volitelná písmena typu:
  - `s` = string
  - `i` = numerická hodnota
  - `b` = boolean
- Používej **pouze anglické názvy**

## Pracovní pravidla (z AGENTS.md / paměti)
- Preferuj udržitelná řešení před chytrými triky.
- Respektuj Filipovy konvence kódu a projektová omezení.
- Veřejná API nech beze změn, pokud není výslovně povoleno jinak.
- Kde to jde, vracej funkční řešení připravená k nasazení.
